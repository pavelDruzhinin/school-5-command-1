# Backend

## Запуск

`cd src && dotnet run`

## Environments

Разные среды нужны чтобы использовались правильные файлы конфигурации

- Development - Для локального тестирования через консоль
- Staging - Для тестирования в докер контейнере
- Production - Для готового приложения
